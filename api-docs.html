Authentication

Every interaction with the Lumix API requires authentication to ensure that only authorized developers and applications are making requests. Our authentication system is based on the use of secure Bearer tokens, which are used to verify the identity of the requesting party and grant access to our API endpoints. To obtain a Bearer token, you'll need to create an account on our platform and follow the instructions provided in the Getting Started guide. Once you have your token, you can use it to authenticate your API requests by including it in the Authorization header of your HTTP requests.

API Endpoints

Lumix provides several powerful endpoints that cover the full spectrum of AI image generation and management. Each endpoint is carefully designed to be simple, flexible, and fast. Below is a comprehensive breakdown of each endpoint with extensive explanations so that developers can get started without guesswork.

1. Generate Image (POST /generate)

This is the core endpoint that allows you to generate AI images from text prompts. It supports multiple AI models, sizes, and generation parameters to give you total creative control.

  • prompt (string, required) – Your description of the image.
  • model (string, optional) – AI model to use. Default: flux-schnell.
  • size (string, optional) – Output size, e.g. 512x512 or 1024x1024.
  • steps (int, optional) – Number of generation steps for quality. Default: 30.

Example Request:

API Request Example

Here's an example of how to make a POST request to the Lumix API:

API Request Example 
POST https://api.lumix.ai/v1/generate
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

{
    "prompt": "A majestic dragon flying over a neon-lit city skyline",
    "model": "realistic-vision-v6",
    "size": "1024x1024",
    "steps": 50
}

Authentication

All requests require an API Key to authorize and secure access to Lumix services. This mechanism protects both your account and our platform from misuse. Make sure to keep your key confidential and never expose it publicly.

The authentication process is straightforward but critical. Upon signing up and logging into the Lumix Dashboard, you'll generate an API Key in the Developer Settings. This key acts as a secret token that you include in your request headers using the Bearer token scheme.

Proper key management is essential. Rotate your API keys periodically and immediately revoke any keys you suspect have been compromised. If you are working in a team environment, create separate keys for different applications or users to maintain security and traceability.

Remember, exposing your API key on the client side (e.g., in frontend JavaScript) risks unauthorized use. Use backend servers or secure environments to handle sensitive keys.

Endpoints

1. Generate Image (POST /generate)

This endpoint allows you to create AI-generated images based on text prompts. You can customize image size, model selection, and generation steps to suit your needs. It's the core functionality powering Lumix's creative capabilities.

When you send a request, the system queues and processes it through the selected model, returning a URL to the generated image. You can embed or download this image in your application.

Advanced users can experiment with different models for unique styles, from realistic visuals to artistic renditions. For large scale use, consider batching requests and caching responses to optimize performance and cost.

Parameters:

  • prompt (string, required) – The text description to generate the image.
  • model (string, optional) – Which AI model to use, defaulting to flux-schnell.
  • size (string, optional) – Desired output resolution, such as 512x512 or 1024x1024.
  • steps (int, optional) – Number of inference steps for quality control; more steps generally improve quality but take longer.

Rate Limits & Best Practices

Lumix API rate limits ensure fair usage and system stability. Free tier users have 60 requests per minute, Pro users 600, and Enterprise plans offer customizable limits.

To maximize efficiency:

  • Cache generated images where possible to reduce redundant calls.
  • Queue or batch requests in high-volume apps to avoid hitting rate limits.
  • Monitor your usage dashboard to proactively manage consumption.

Consider exponential backoff for handling 429 (Too Many Requests) or 500 (Server Error) responses gracefully.

Error Handling

The Lumix API uses standard HTTP status codes to indicate success or failure of API requests. All error responses return a JSON body with detailed messages to help you diagnose and fix issues.

  • 200 OK – Request was successful.
  • 400 Bad Request – The request was malformed or missing required fields.
  • 401 Unauthorized – Your API key is invalid or missing.
  • 429 Too Many Requests – You have exceeded your rate limit; slow down your requests.
  • 500 Server Error – An unexpected error occurred on the server; try again later.

Best Practice: Implement exponential backoff retries when you receive 429 or 500 status codes to improve reliability.
