Lumix Velocity

Ultra-fast AI image generation designed for rapid prototyping & high-throughput applications.

Model Overview

Lumix Velocity is a cutting-edge AI model optimized for extremely fast image generation. Built on a streamlined architecture, it balances speed and visual quality by leveraging efficient transformer backbones and custom pruning techniques.

This model is perfect for developers and businesses who require near real-time responses without compromising too much on image fidelity. It supports batch processing and is designed to scale horizontally across cloud infrastructure.

API Endpoint

POST https://api.lumix.ai/v1/generate

Use this endpoint to generate images with Lumix Velocity by specifying the model in the request payload.

Request Headers

  • Authorization: Bearer <your_api_key> — Your API authentication token.
  • Content-Type: application/json

Request Body Parameters

{
  "prompt": "string (required)",
  "model": "lumix-velocity",
  "size": "string (optional, default: 512x512)",
  "steps": "integer (optional, default: 20)",
  "guidance_scale": "float (optional, 1.0-20.0, default: 7.5)",
  "seed": "integer (optional, for reproducibility)",
  "batch_size": "integer (optional, default: 1, max: 8)"
}

Sample API Request

{
  "prompt": "A futuristic cityscape at dusk, vibrant neon lights",
  "model": "lumix-velocity",
  "size": "512x512",
  "steps": 20,
  "guidance_scale": 7.5,
  "seed": 12345,
  "batch_size": 2
}

Sample API Response

{
  "status": "success",
  "request_id": "req_abc123",
  "images": [
    "https://cdn.lumix.ai/generated/futuristic_city_123_1.jpg",
    "https://cdn.lumix.ai/generated/futuristic_city_123_2.jpg"
  ],
  "model_used": "lumix-velocity",
  "processing_time": "1.8s"
}

Authentication & Rate Limits

  • All requests require a valid API key via the Authorization header.
  • Rate limit: 60 requests per minute per API key.
  • Batch requests count as multiple requests based on batch_size.

Error Handling

API returns standard HTTP status codes. Common errors include:

  • 401 Unauthorized — Invalid or missing API key.
  • 429 Too Many Requests — Rate limit exceeded.
  • 400 Bad Request — Invalid payload or parameters.

Always check the message field in the JSON response for detailed info.

Tips & Best Practices

  • Use concise prompts for faster processing.
  • Leverage seed parameter for reproducible images.
  • Batch multiple images in one request with batch_size to optimize throughput.
  • Cache frequently used images on your side to reduce API calls.
  • Combine Lumix Velocity with other models like Lumix Precision for detailed results when needed.
