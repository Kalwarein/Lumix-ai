Lumix Precision

High-fidelity AI image generation model for ultra-realistic and detailed outputs.

Model Overview

Lumix Precision leverages advanced diffusion and transformer architectures to produce ultra-realistic, detailed images ideal for professional artists, designers, and content creators. It prioritizes image quality and fine detail over generation speed.

This model excels at rendering complex textures, intricate lighting, and natural environments, making it perfect for high-end visual projects and print-ready graphics.

API Endpoint

POST https://api.lumix.ai/v1/generate

Specify lumix-precision as the model in the payload to access this model’s capabilities.

Request Headers

  • Authorization: Bearer <your_api_key>
  • Content-Type: application/json

Request Body Parameters

{
  "prompt": "string (required)",
  "model": "lumix-precision",
  "size": "string (optional, default: 512x512)",
  "steps": "integer (optional, default: 50)",
  "guidance_scale": "float (optional, 1.0-20.0, default: 9.0)",
  "seed": "integer (optional)",
  "batch_size": "integer (optional, default: 1, max: 4)"
}

Sample API Request

{
  "prompt": "A hyper-realistic portrait of a young woman with detailed skin textures",
  "model": "lumix-precision",
  "size": "512x512",
  "steps": 50,
  "guidance_scale": 9.0,
  "seed": 67890,
  "batch_size": 1
}

Sample API Response

{
  "status": "success",
  "request_id": "req_xyz789",
  "images": [
    "https://cdn.lumix.ai/generated/portrait_young_woman_789.jpg"
  ],
  "model_used": "lumix-precision",
  "processing_time": "4.5s"
}

Authentication & Rate Limits

  • Valid API key via Authorization header required.
  • Rate limit: 30 requests per minute per API key.
  • Batch requests count as multiple requests based on batch_size.

Error Handling

Common HTTP errors:

  • 401 Unauthorized — Invalid/missing API key.
  • 429 Too Many Requests — Rate limit exceeded.
  • 400 Bad Request — Invalid payload.

Check JSON message field for details.

Tips & Best Practices

  • Use detailed, descriptive prompts for best quality.
  • Increase steps for finer detail at cost of speed.
  • Use guidance_scale to balance creativity vs. adherence to prompt.
  • Batch smaller numbers (max 4) to avoid latency spikes.
  • Cache frequently used images locally when possible.
